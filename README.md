# Local AI Second Brain Server

> VSCode 코딩 비서와 Obsidian을 연결하는 **로컬 자동 지식 관리 시스템**입니다.
> 개발자가 직접 정리하지 않아도, AI가 알아서 대화를 걸러내고 위키 문서를 만들어 줍니다.

---

## 왜 이걸 만들었나요?

개발을 하다 보면 두 가지 상황이 반복됩니다.

1. **코딩 중 AI에게 물어본 유용한 내용** → 채팅창이 닫히면 영영 사라짐
2. **웹 서핑 중 스크랩한 파편화된 정보들** → Obsidian에 넣어두긴 했는데 정리는 나중으로 미룸

이 프로젝트는 이 두 가지를 자동으로 해결합니다.
평소처럼 VSCode에서 코딩하거나, 메모를 폴더에 던져두기만 하면 됩니다.
나머지는 백그라운드에서 AI 두 개가 알아서 처리합니다.

---

## 어떻게 동작하나요?

크게 두 가지 입력 경로가 있습니다.

### 경로 1 — VSCode에서 AI에게 질문할 때

```
VSCode (Continue 확장) → Node.js 서버 → Qwen (코드 생성)
                                       ↘ Gemma (가치 판단) → Obsidian Wiki
```

- VSCode에서 Continue 확장을 통해 AI에게 질문합니다.
- **Qwen**이 답변을 스트리밍으로 즉시 돌려줍니다. 응답 속도에 지연이 없습니다.
- 답변이 끝난 뒤, **Gemma**가 백그라운드에서 조용히 판단합니다.
  - "이건 그냥 오타 고친 거잖아?" → 버림 (`SKIP`)
  - "이건 아키텍처 설계 내용이네" → Obsidian에 위키 문서로 저장 (`SAVE`)

### 경로 2 — 메모나 스크랩을 폴더에 넣을 때

```
1_Raw_Inbox (파일 투척) → Node.js 파일 감시자 → Qwen (문서 정제)
                                                → 2_Wiki_Auto (완성 문서)
                                                → 3_Raw_Archive (원본 보관)
```

- `1_Raw_Inbox` 폴더에 텍스트 파일, 로그, 웹 클립 등을 그냥 던져두면 됩니다.
- 서버가 파일 생성을 감지하고, Qwen이 내용을 분석하여 목차와 요약이 갖춰진 마크다운 위키로 변환합니다.
- 완성된 문서는 `2_Wiki_Auto`로, 원본은 `3_Raw_Archive`로 자동 이동됩니다.

## 핵심 특징 4가지

| 특징 | 설명 |
|---|---|
| **⚡ 지연 없는 응답** | 서버가 Qwen으로 직접 프록시하므로 응답 스트리밍 속도에 영향 없음 |
| **🚦 AI 수문장** | Gemma가 대화를 평가해서 저장할 가치가 있는 것만 골라냄 |
| **📝 자동 위키 생성** | 난잡한 메모를 구조화된 마크다운 문서로 자동 변환 |
| **📂 안전한 파일 처리** | 서버 재시작 시 누적 파일 자동 처리, 대량 파일은 큐로 순차 처리 |

## 사용 모델 구성

| 역할 | 모델 | 실행 방식 | 담당 업무 |
|---|---|---|---|
| **메인 엔진** | Qwen 3.6 | vLLM (데스크탑) | 코드 생성, 문서 정제 |
| **백그라운드 사서** | Gemma 4 | Ollama (노트북) | 대화 가치 평가, 태그/요약 생성 |

> 두 기기를 사용하는 분산 구성입니다. 단일 기기로도 모두 실행 가능합니다.

## 시작하기

### 1단계 — 환경 변수 설정

루트 디렉토리에 `.env` 파일을 만들고 아래 내용을 채워 넣습니다.

```env
# 서버 포트
PORT=3000

# Obsidian Vault 경로 (절대 경로)
OBSIDIAN_BASE_PATH=C:/Users/이름/Documents/MyVault

# Qwen 서버 주소 (vLLM)
VLLM_BASE_URL=http://192.168.0.10:8000/v1
VLLM_QWEN_MODEL=qwen2.5-coder:32b

# Gemma 서버 주소 (Ollama)
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_GEMMA_MODEL=gemma2:9b
```

### 2단계 — 설치 및 실행

```bash
# 패키지 설치
npm install

# 서버 시작
node server.js
```

## 실제 사용 시나리오

**시나리오 A — 코딩 중 자동으로 지식 쌓기**

1. VSCode에서 복잡한 로직 리팩토링을 AI에게 요청합니다.
2. Qwen이 답변을 스트리밍하는 동안, Gemma가 해당 대화를 분석합니다.
3. 대화 종료 후 `2_Wiki_Auto/리팩토링_전략_20260520.md`가 자동으로 생성됩니다.
4. 내가 한 일: 그냥 평소처럼 AI에게 물어봤을 뿐.

---

**시나리오 B — 웹 스크랩 자동 정리**

1. 웹 서핑 중 유익한 내용을 텍스트 파일로 복사해 `1_Raw_Inbox`에 저장합니다.
2. 서버가 파일을 감지하고 정제를 시작합니다.
3. `2_Wiki_Auto` 폴더에 목차와 요약이 갖춰진 문서가 도착합니다.
4. 내가 한 일: 파일 하나 폴더에 던진 것 뿐.
